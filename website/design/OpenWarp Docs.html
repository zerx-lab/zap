Introduction

介绍

OpenWarp is a community fork of Warp that keeps the full terminal experience and opens up the AI layer. This page covers what it is, what's different from upstream, and where to go next.

OpenWarp 是 Warp 的社区 fork,完整保留终端体验,把 AI 层完全打开。这一页讲清楚:它是什么、与上游的差异、接下来该读什么。

What is OpenWarp?

OpenWarp 是什么

OpenWarp is a Rust-based terminal that reuses the upstream Warp UI — Blocks, Workflows, command suggestions, the keymap layer — and replaces the closed AI gateway with explicit, local, editable plumbing. It is not the official Warp product, and it is not affiliated with Warp Inc.

OpenWarp 是基于 Rust 的终端,复用上游 Warp 的 UI ── Blocks、Workflows、命令建议、keymap 层 ── 把封闭的 AI gateway 替换成显式、本地、可编辑的管线。它不是官方 Warp,也与 Warp Inc. 无任何关联。

Why a fork?

为什么 fork

Upstream Warp routes AI traffic through its own gateway. That works for most people, but it doesn't work for users who want to bring their own keys, point at a self-hosted model, or audit the system prompt that's getting sent on their behalf. OpenWarp changes five things; everything else stays.

上游 Warp 把 AI 流量经过自己的 gateway。对大部分人没问题,但对要自带 key、指向自托管模型、审计系统提示词的用户来说不够。OpenWarp 只改 五件事,其他全留。

Install

安装

There are no binary installers yet — OpenWarp is in early development. Build from source via either the bootstrap script or cargo directly.

目前没有发布安装包 ── OpenWarp 处于早期开发阶段。请从源码构建,推荐使用 bootstrap 脚本,也可以直接用 cargo。

# Clone & bootstrap (recommended)
git clone https://github.com/zerx-lab/warp.git openwarp
cd openwarp

# installs deps, builds dev profile, launches
./script/run
# OSS users only need this binary
git clone https://github.com/zerx-lab/warp.git openwarp
cd openwarp

cargo run --release --bin warp-oss
OSS users: --bin warp-oss is the binary without any cloud-bound features compiled in. It's the right target for self-hosters and most contributors. OSS 用户:--bin warp-oss 是不带任何云端能力的 binary,自托管和大多数贡献者都用这个。

First run

首次运行

After launch, open Settings → Agents → Providers and add at least one provider. The minimal config looks like this:

启动后,打开 设置 → Agents → 提供商,至少添加一个 provider。最小配置如下:

~/.config/openwarp/settings.toml TOML 
# Pick any OpenAI-compatible endpoint
[providers.deepseek]
base_url = "https://api.deepseek.com/v1"
protocol = "deepseek"     # or: openai, anthropic, gemini, ollama
api_key  = "sk-•••"          # stored locally, never sent home

[[providers.deepseek.models]]
display = "DeepSeek V4 Pro"
id      = "deepseek-v4-pro"
context = 1000000
output  = 384000

Hit ⌘P to switch model, / to open the slash menu, ⌃⇧+ to enter code review.

按 ⌘P 切换模型,/ 打开 slash 菜单,⌃⇧+ 进入代码评审。

SSH manager

SSH 管理器

OpenWarp adds a first-class SSH manager — net-new on top of upstream. Hosts live in their own file, jump-host chains are supported, and the remote session renders as Blocks just like a local shell.

OpenWarp 在上游基础上新增了一等公民的 SSH 管理器。主机定义放在独立文件,支持跳板机链路,远端会话同样以 Blocks 形式呈现,体验与本地 shell 一致。

~/.config/openwarp/ssh.toml TOML 
[host.bastion]
user = "ops"
host = "bastion.example.com"
port = 22

[host.prod-edge]
user = "deploy"
host = "edge-01.lan"
jump = "bastion"           # chain through bastion
forward_agent = true
SSH credentials are read from your existing ~/.ssh — OpenWarp does not duplicate them or upload them anywhere. The ssh.toml file holds connection metadata only. SSH 凭证沿用你现有的 ~/.ssh ── OpenWarp 不复制、不上传。ssh.toml 只保存连接元数据。

Localization

本地化

English and Simplified Chinese ship in-tree. The locale switcher is in Settings → Appearance → Language; restart OpenWarp for the change to take full effect.

英文与简体中文随发行包一起提供。语言切换位于 设置 → 外观 → 语言;切换后需要重启 OpenWarp 完全生效。
