[ HOW ]

How OpenWarp handles every prompt.

每一次对话,OpenWarp 是这样处理的.

OpenWarp keeps the upstream Warp UI — Blocks, Workflows, command suggestions — and replaces the closed AI gateway with explicit, local, editable plumbing.

OpenWarp 完整保留上游 Warp 的 Blocks / Workflows / 命令建议,只把封闭的 AI gateway 换成显式、本地、可编辑的管线。

3 stages 3 阶段
STAGE 01
Captures your block
捕获你的 Block
Each command becomes a Block with its cwd, env, exit code — the full unit of context.
每条命令封装成 Block,带上 cwd / 环境 / 退出码 ── 完整的上下文单元。
STAGE 02
Routes to your provider
路由到你的 Provider
Your settings.toml picks the model. The request goes straight from your laptop to the endpoint — no middle hop.
由 settings.toml 决定走哪个模型。请求直接从你的设备打到端点,没有中间一跳。
STAGE 03
Streams back into Blocks
流式回写到 Block
SSE tokens land in the same Block UI you already know. Tools execute locally and re-enter the loop.
SSE token 流式回到熟悉的 Block UI,工具在本地执行后重新回到循环。

想接什么 provider,就接什么.

Native protocols, OpenAI-compatible endpoints, self-hosted Ollama — configured by editing one TOML file.

原生协议、OpenAI 兼容端点、自托管 Ollama ── 编辑一个 TOML 文件就能切换。

[ 02 / 04 ]

Adapts to your terminal.

完全 服从 你的终端.

Custom rules. Custom prompts. Custom roles. The AI side of OpenWarp is your house — not ours.

规则你写、prompt 你改、角色你切。OpenWarp 的 AI 层只是房子,屋主始终是你。

Bring your own provider
自带 Provider

指向任意 OpenAI 兼容端点,或直接走 DeepSeek / Anthropic / Gemini / Ollama 的原生协议。

[ 03 / 04 ]

A first-class SSH manager.

一等公民的 SSH 管理.

Net-new in OpenWarp on top of upstream Warp. Saved hosts, jump-host chains, Block-aware remote sessions — all backed by your existing ~/.ssh.

OpenWarp 在上游基础上新增的能力。保存主机、跳板链、Block 感知的远端会话 ── 全部沿用你现有的 ~/.ssh。

net-new 本仓新增
01
Saved hosts
主机收藏
Pin frequent hosts. Connect with one keystroke.
收藏常用主机,一次按键即连接。
02
Jump chains
跳板链
Multi-hop routing through a bastion declared in ssh.toml.
通过 ssh.toml 声明的多跳路由。
03
Block-aware remote
Block 感知远端
Remote sessions render as Blocks — same UX as local.
远端会话同样以 Block 呈现,体验一致。
04
Keys stay home
密钥不离开本机
Existing ~/.ssh reused. ssh.toml holds metadata only.
沿用你的 ~/.ssh,ssh.toml 只保存元数据。

[ FAQ ]

Frequently asked.

常见问题.

Is this affiliated with Warp Inc.?这是 Warp 官方的吗?+
No. OpenWarp is a community fork. We are not affiliated with, endorsed by, or sponsored by Warp Inc. The trademarks belong to their respective owners.
不是。OpenWarp 是社区 fork,与 Warp Inc. 无任何隶属、背书或赞助关系。商标归各自所有者持有。
Can I use my company's private model?能接公司内部的私有模型吗?+
Yes — if it speaks any of the supported native protocols or exposes an OpenAI-compatible Chat Completions endpoint, just point base_url at it.
可以 ── 只要支持任一原生协议,或暴露 OpenAI 兼容的 Chat Completions 端点,把 base_url 指过去就行。
What's different from upstream Warp?和上游 Warp 有什么差别?+
Five things: AI provider (any vs cloud gateway), credentials (local vs cloud account), prompts (editable vs server-assembled), SSH (built-in manager vs none), cloud features (off vs on by default). Everything else stays.
五件事:AI provider(任意 vs 云端 gateway)、凭证(本地 vs 云端账号)、prompt(可编辑 vs 服务端组装)、SSH(内置管理器 vs 没有)、云端能力(默认关 vs 默认开)。其他全部保留。
Is it production-ready?能上生产吗?+
Not yet. v0.x — expect breaking changes, rough edges, and thin docs. File issues, send PRs.
暂时不行。v0.x 阶段会有 breaking change、粗糙之处、文档不全。欢迎提 issue 与 PR。
License?许可证?+
Source-available, inheriting the upstream Warp license terms. Commercial redistribution may require a separate agreement — check LICENSE.md.
Source-available,沿用上游 Warp 的许可条款。商业重分发可能需要单独协议 ── 详见 LICENSE.md。
